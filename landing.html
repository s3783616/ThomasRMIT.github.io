Project

I aim to increase security for systems and users. It will be a program that will defend a system from intrusion, stop personal data from being mined by websites; and educate the user to be more wary of social engineering attempts. This project will help me gain insight and knowledge into the field of cybersecurity and machine learning; as well as getting me acquainted with the social engineering aspect of the profession. Through this, I'll have some of the necessary skills to join ASIS, and protect against intrusion by hostile parties. It shall be called... STEVE

Information privacy, and protecting against surveillance is becoming an increasing concern nowadays. With so many websites taking data from their users, and corporations and governments spying on the people for their own sinister purposes; I feel that there must be something protecting society's data from being manipulated for evil purposes. I vow to create this shield! Machine Learning seems like a good way for a computer to recognise attacks, and react in programmed ways to counter-act them. It's also an interesting approach to the problem!

The program will be equipped with a library of known scams and tactics, as well as the ability to recognise them, and alert the user to what the detected intrusion attempt is doing, and provide instruction on how to stop it. It will also have the ability to scan any page, form or application that requires the user give their personal data to. If STEVE finds certain issues with the page in question, such as bad grammar or references to international banks such as Western Union; it will alert the user.

It will be difficult to make it work with the accuracy that I want, detecting the threats and warning the user; but that's not it's only function. It educates the user on the common tactics used by social engineers and phishers.

On the design side, it should be user friendly, as making security accessible is a major part of why I want to make STEVE. If the user can't *use* it, then the project has failed before it's begun. It will contain a library of common tactics and schemes that scammers, companies, and governments use to obtain personal information, and what they do with it. This should be easy to read and understand. My goal is to educate users to question whether or not they should trust something with their data. That is impossible if they don't understand what they should be on the lookout for. STEVE can only do so much; and although I want it to defend against everything, it should eventually fall to the user to determine what they trust, or don't trust. STEVE will warn the user, and stop malware from being executed, but the user always makes the decision to trust. STEVE, as an app, could run on a computer or smartphone. Since it's software, I imagine the actual code will be all that's required, so long as it has something to run on. I'll need something to create it with though, perhaps using a programming language like C++

I'll need to learn a lot about how data is gathered, how and why it can be used maliciously, and compile that in an easy-to-read, educational format. Machine learning is a complicated subject, which I know nothing about. I've also got to learn a coding language fluently. Thus, I must learn about that too. As for feasibility, it's a great deal of complicated topics to cover, but I imagine I can do it. Eventually. Give or take 5 years.

If STEVE has the impact I intend, society will become more security conscious, both online and offline. Their data will be secure, at least until people inevitably come up with cleverer and more refined tactics. Companies will have their information stolen less, governments will have a more security conscious workforce, preventing hacks and leaks, and the general public will enjoy a life with a lessened risk of having their data stolen.
